# Support

SSH Onboard is maintained as an open-source project.

- Usage questions and reproducible bugs: open a [GitHub issue](https://github.com/ZhangZhengruiNUS/ssh-onboard/issues) with the extension version, VS Code version, Windows version, and sanitized diagnostics.
- Feature proposals: start a GitHub issue describing the user problem and expected workflow.
- Security vulnerabilities: follow [SECURITY.md](SECURITY.md). Do not disclose them in a public issue.

Never attach passwords, private keys, full public keys, `authorized_keys`, or an unredacted SSH configuration. Use **SSH Onboard: Show Diagnostics** when possible.
